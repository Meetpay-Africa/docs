title: "Merchant default URLs"
description: "PUT /api/v1/merchants/me"
---

Optional defaults when you do not send URLs on each request.

`PUT /api/v1/merchants/me`

```json
{
  "webhook_url": "https://api.example.com/paygrid/webhooks",
  "callback_url": "https://api.example.com/paygrid/callback"
}
```

Required for **payout** notifications (payout create does not accept URL fields). For **payments**, you can use these defaults or pass `webhook_url` / `callback_url` on `POST /api/v1/payments` instead.

See [Webhooks](/guides/webhooks).

<Card title="API reference" icon="code" href="/api-reference/merchants/update-profile">
  PUT /api/v1/merchants/me
</Card>

```bash
curl -X PUT "https://meet.briq.tz/api/v1/merchants/me" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"webhook_url":"https://api.example.com/paygrid/webhooks","callback_url":"https://api.example.com/paygrid/callback"}'
```
